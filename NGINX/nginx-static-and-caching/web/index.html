Static Content & NGINX Reverse Proxy

Static content refers to files that do not change in response to user requests. These files are delivered to the client exactly as they are stored on the server. Common examples include: HTML files, CSS files, JavaScript files, Images (JPEG, PNG, GIF), Videos. Since static files donâ€™t require server-side processing, they can be served much more quickly than dynamic content generated in real-time.
NGINX can be configured to function as a reverse proxy. A reverse proxy acts as an intermediary server that sits in front of one or more backend servers and forwards client requests to them. It then receives the responses from the backend servers and delivers them back to the client.
